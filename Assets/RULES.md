# Unity Study Rules

## 1. 계획부 (Planning)
- 이 단계에서는 **절대로 코드를 작성하지 않습니다.**
- 구현할 기능, 구조, 사용할 에셋이나 리소스 등을 계획합니다.
- 요구사항이 불명확하거나 부족한 부분이 있다면 **추가 질문**을 통해 명확히 합니다.
- 전체적인 로직의 흐름이나 아키텍처를 구상합니다.

## 2. 구현부 (Implementation)
- 계획된 내용을 바탕으로 실제 코드를 작성하고 기능을 구현합니다.
- 구현이 완료된 후에는 다음 내용을 반드시 포함해야 합니다:
    - **사용 기술 및 이유**: 왜 해당 기술(API, 패턴 등)을 사용했는지 설명합니다.
    - **이점**: 그 기술을 사용함으로써 얻을 수 있는 이점(성능, 가독성, 유지보수성 등)을 설명합니다.

## 3. 요약 (Summary)
- 모든 작업의 마지막에는 항상 **짧은 요약**을 제시합니다.
- 진행한 내용의 핵심을 간략하게 정리합니다.
